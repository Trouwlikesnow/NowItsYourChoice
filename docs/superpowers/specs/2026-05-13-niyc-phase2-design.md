# NowItsYourChoice 阶段 2 设计文档：总资产看板 + 持仓更新

**项目名**：NowItsYourChoice（缩写 NIYC）
**日期**：2026-05-13
**阶段**：Phase 2
**预计周期**：1-2 周

---

## 1. 阶段目标

在阶段 1（决策引擎 + 关注板块 MVP）的基础上，新增持仓管理和资产看板能力：

- 用户在飞书给 Bot 发送券商成交截图，系统自动识别并录入交易记录
- 每日自动根据交易记录更新持仓、刷新市值、生成资产快照
- 通过 Bitable 仪表盘提供 5 个可视化看板

### 范围内

- 截图识别 → 交易记录自动录入
- 持仓表自动维护（数量、成本、市值、浮盈）
- 每日资产快照生成（总市值、总浮盈、当日盈亏）
- 仓位占比预警（超 20% 标红，对应交易纪律）
- 5 个 Bitable 仪表盘看板

### 不在范围内

- 基金持仓（仅 A 股）
- 分钟级实时行情
- 券商 App 直连
- 操作复盘分析（阶段 4）

## 2. 技术架构

### 2.1 架构总图

```
实时链路（交易录入）：
  用户发截图 → 飞书 Bot 消息
    → 飞书事件订阅（Webhook）
    → Cloudflare Worker
      → 获取 tenant_access_token
      → 下载图片（飞书 Open API）
      → 调用多模态模型 API（GPT-4o）识别交易内容
      → 写入 Bitable「交易记录表」
      → Bot 回复用户确认消息
    
每日链路（GitHub Actions，15:30 北京时间）：
  复用阶段 1 的 daily.yml，扩展 main.py：
    → 阶段 1 原有逻辑（指标、快照、新闻）
    → 读取交易记录表 + 持仓表 → 重算持仓
    → 用最新收盘价刷新持仓市值/浮盈
    → 生成当日资产快照
    → 仓位占比预警检查
```

### 3.2 新增表：持仓表

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 股票代码 | 文本 | 主键 |
| 股票名称 | 文本 | |
| 所属板块 | 关联 | 链接板块表 |
| 持仓数量 | 数字 | 由交易记录累计计算 |
| 成本价 | 数字 | 加权平均成本 |
| 成本金额 | 数字 | = 持仓数量 × 成本价 |
| 当前价 | 数字 | 每日由 GitHub Actions 刷新 |
| 市值 | 数字 | = 持仓数量 × 当前价 |
| 浮盈额 | 数字 | = 市值 - 成本金额 |
| 浮盈% | 数字 | = 浮盈额 / 成本金额 × 100 |
| 仓位占比% | 数字 | = 本股市值 / 总市值 × 100 |
| 仓位预警 | 单选 | 正常 / 超标 |

**成本价计算规则（券商摊薄成本逻辑）**：

与券商 App 显示一致，手续费计入成本：

- **买入**：成本金额 = 原成本金额 + 买入金额 + 手续费合计
  - 成本价 = 成本金额 / 总数量
- **卖出**：成本金额 = 原成本金额 - 卖出金额 + 手续费合计
  - 成本价 = 新成本金额 / 剩余数量
  - 卖出盈利时成本价会降低，亏损时会升高（与券商一致）
- **清仓**（持仓归零）：保留记录，持仓数量设 0

示例：
| 操作 | 数量 | 价格 | 手续费 | 成本金额 | 成本价 |
|------|------|------|--------|---------|--------|
| 买入 1000股 | 1000 | 10.00 | 30 | 10,030 | 10.03 |
| 卖出 500股 | 500 | 12.00 | 25 | 10,030 - 6,000 + 25 = 4,055 | 8.11 |

**手续费计算规则**：

截图能识别到手续费就用识别值，识别不到按以下费率自动估算：

| 费用 | 招商证券 | 华泰证券 | 适用 |
|------|---------|---------|------|
| 佣金 | 万 2.5（0.025%），最低 5 元 | 万 1（0.01%），最低 5 元 | 买卖都收 |
| 印花税 | 0.05% | 0.05% | 仅卖出 |
| 过户费 | 0.001% | 0.001% | 买卖都收 |

交易记录中通过「券商」字段区分费率。用户可在 Bitable 中手动修正任意手续费字段。

## 4. Cloudflare Worker：截图识别服务

### 4.1 职责

一个 Worker，处理飞书 Bot 消息事件：

1. **事件验证**：处理飞书 challenge 验证请求 + 签名校验
2. **图片获取**：从消息事件中提取 image_key，调飞书 API 下载图片
3. **AI 识别**：将图片 base64 发给 GPT-4o，提取交易记录
4. **数据写入**：调 Bitable API 写入交易记录表
5. **消息回复**：通过飞书 Bot API 回复确认消息

### 4.2 多模态识别 Prompt

```
你是一个券商成交记录识别助手。请从这张截图中提取所有成交记录。

要求：
1. 识别每笔交易的：股票代码、股票名称、买卖方向、成交价格、成交数量、成交金额
2. 如果能识别到佣金、印花税、过户费等手续费明细，也提取出来
3. 如果能识别出是哪家券商的 App 界面（如招商证券、华泰证券），填入 broker 字段
4. 如果无法识别某个字段，设为 null
5. 如果截图不是成交记录，返回 {"error": "非成交记录截图"}

严格按以下 JSON 格式返回，不要附加任何说明文字：
{
  "broker": "招商证券",
  "trades": [
    {
      "code": "002594",
      "name": "比亚迪",
      "direction": "买入",
      "price": 285.50,
      "quantity": 500,
      "amount": 142750.00,
      "commission": 35.69,
      "stamp_tax": 0,
      "transfer_fee": 1.43,
      "time": "2026-05-13 14:30:00"
    }
  ]
}
```

### 4.3 环境变量

| 变量名 | 说明 |
|--------|------|
| FEISHU_APP_ID | 飞书应用 ID |
| FEISHU_APP_SECRET | 飞书应用 Secret |
| FEISHU_VERIFICATION_TOKEN | 事件订阅验证 token |
| FEISHU_ENCRYPT_KEY | 事件加密 key（可选） |
| FEISHU_BASE_APP_TOKEN | Bitable app token |
| TABLE_ID_TRADES | 交易记录表 ID |
| LLM_API_BASE | 中转站 API 地址 |
| LLM_API_KEY | 中转站 API Key |
| LLM_MODEL | 模型名，默认 gpt-4o |

### 4.4 错误处理

- 图片下载失败 → 回复用户「图片获取失败，请重新发送」
- AI 识别返回 error → 回复用户「无法识别，请确认是成交记录截图」
- AI 返回的 JSON 格式异常 → 回复用户「识别异常，请手动录入」
- Bitable 写入失败 → 回复用户「写入失败，请稍后重试」
- 所有错误不阻塞 Worker，返回 200 给飞书（避免重试风暴）

### 4.5 回复格式

识别成功后，Bot 回复：

```
✅ 已录入 2 笔交易（招商证券）：

1. 买入 比亚迪(002594) 500股 × 285.50 = 142,750.00  手续费 37.12
2. 卖出 宁德时代(300750) 200股 × 210.30 = 42,060.00  手续费 31.87

如有误差请在多维表格中修改。
```

## 5. GitHub Actions 每日任务扩展

### 5.1 新增模块：`scripts/portfolio.py`

职责：

1. **同步持仓**：读取交易记录表中今日新增记录，更新持仓表（券商摊薄成本逻辑）
   - 买入 → 增加数量，成本金额 += 买入金额 + 手续费，重算成本价
   - 卖出 → 减少数量，成本金额 -= 卖出金额 - 手续费，重算成本价
   - 新股票 → 创建持仓记录
   - 清仓 → 持仓数量设 0

2. **刷新市值**：用标的表中已更新的最新收盘价刷新持仓表
   - 市值 = 持仓数量 × 当前价
   - 浮盈额 = 市值 - 成本金额
   - 浮盈% = 浮盈额 / 成本金额 × 100

3. **仓位预警**：计算每只股票的仓位占比
   - 仓位占比% = 本股市值 / 总市值 × 100
   - 超过 20% 标记为「超标」

4. **生成快照**：汇总所有持仓，写入资产快照表一行

### 5.2 执行顺序

```python
# main.py 扩展后的执行顺序：
def main():
    # --- 阶段 1 原有逻辑 ---
    for ticker in tickers:
        process_ticker(ticker, cfg, bitable)      # 指标 + 快照 + 刷新标的
    for sector in sectors:
        fetch_and_store_news(sector, cfg, bitable) # 板块新闻
    cleanup_rolling_window(cfg, bitable)            # 清理过期数据
    
    # --- 阶段 2 新增逻辑 ---
    sync_portfolio(cfg, bitable)    # 同步交易记录 → 更新持仓
    refresh_market_values(cfg, bitable)  # 刷新持仓市值/浮盈
    check_position_alerts(cfg, bitable)  # 仓位预警
    take_asset_snapshot(cfg, bitable)    # 生成当日资产快照
    cleanup_asset_snapshots(cfg, bitable) # 清理 >90 天的资产快照
```

### 5.3 持仓初始化（基线数据）

首次使用时需要设置基线持仓。两种方式：

- **方式 A**：直接在 Bitable 持仓表里手动填入当前持仓（代码、数量、成本价）
- **方式 B**：发截图录入一笔虚拟的「基线买入」记录

推荐方式 A，一次性手动填完最准确。

## 6. Bitable 仪表盘看板

5 个看板全部在 Bitable 仪表盘中手动配置，不需要代码。

### 6.1 板块分组持仓

- **数据源**：持仓表
- **视图类型**：分组表格
- **分组字段**：所属板块
- **展示字段**：股票名称、持仓数量、市值、浮盈%、仓位占比%
- **排序**：组内按市值降序

### 6.2 收益排行榜

- **数据源**：持仓表
- **视图类型**：表格视图
- **排序**：浮盈% 降序
- **展示字段**：股票名称、成本价、当前价、浮盈额、浮盈%
- **条件格式**：浮盈% > 0 绿色，< 0 红色

### 6.3 仓位雷达图

- **数据源**：持仓表
- **视图类型**：饼图
- **数值字段**：市值
- **分类字段**：股票名称
- **辅助**：旁边放一个筛选视图，只显示仓位预警 = 超标的记录

### 6.4 每日盈亏曲线

- **数据源**：资产快照表
- **视图类型**：折线图
- **X 轴**：日期
- **Y 轴**：总市值（主轴）、当日盈亏（副轴）
- **时间范围**：最近 90 天

### 6.5 近期交易流水

- **数据源**：交易记录表
- **视图类型**：表格视图
- **排序**：交易时间降序
- **展示字段**：交易时间、股票名称、方向、成交价、成交数量、成交金额
- **筛选**：最近 30 天
- **条件格式**：买入蓝色，卖出橙色

## 9. 验收标准

### 9.1 截图识别

- [ ] 给飞书 Bot 发一张券商成交截图，Bot 在 10 秒内回复识别结果
- [ ] 交易记录表中自动新增对应记录，字段正确

### 9.4 仓位预警

- [ ] 单股仓位超过 20% 时，仓位预警字段标记为「超标」

- [ ] 电脑和手机都能查看
